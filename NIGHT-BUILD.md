## 0) 60초 요약
- 지금 단계(사용자 ~126명)의 진짜 병목은 **유통 + 한 가지로 각인**. 그래서 **"매일 아침 3분 브리핑"을 간판 상품**으로, **스레드**를 주력 유통 채널로 잡았다.
- 포지셔닝: **"오늘의 이슈, 돈으로 읽다"** — 핫토픽을 *돈/생활경제 렌즈*로. broad하지만 온브랜드·안전·사이트 전환됨.
- 오늘 밤 만든 것: ① 브리핑 랜딩 `/briefing/` ② 스레드 드래프트 자동생성기 ③ 발행 파이프라인 통합 ④ **즉시 게시 가능한 스레드 초안 7개**.
- **너가 아침에 결정할 것 3가지**는 맨 아래 ✅ 섹션에.

## 3) 성장 플레이북 (founder 모드 — 트래픽·모객·수익화)

### A. 트래픽 — 스레드 올인 (지금 1순위)
- **왜 스레드**: 텍스트 친화·초기 도달 좋음·대화형. AI 블로그는 SEO 느려서, 소셜이 더 빠른 도달.
- **콘텐츠**: 매일 발행 글 → `gen-threads.js`가 "돈으로 읽다" 초안 자동 생성 → **복붙해서 게시**.
- **루틴**: 매일 아침 1개(브리핑) + 낮에 1개(아카이브 글 재활용, `--batch`로 미리 뽑아둠). **주 5~7회**.
- **목표 지표**: 페이지뷰❌ → **스레드 팔로워 0→1000** + **푸시/뉴스레터 구독자 수**.

### B. 모객(전환) — 소유 청중으로
- 스레드/검색 유입 → **`/briefing/`에서 푸시 구독** 전환이 핵심 (이미 깔린 Web Push 활용).
- 친구 초대 +20밀웜(기존)·업적 자랑하기(소셜 바이럴) 이미 있음 → 스레드 유입과 연결.
- **추천 다음 수**: 글 하단/브리핑에 "매일 아침 받기" 구독 CTA를 더 강하게(현재도 있음).

### C. 수익화 — 브랜드('팔지 않는다') 지키며
- 애드센스·멤버십 이미 셋업됨. 지금은 **수익화보다 트래픽이 먼저** (광고는 트래픽의 함수).
- 트래픽 1000명대 오면: 멤버십 가치 강화 + 계산기 내 제휴(affiliate) 자연스럽게. **지금 광고 늘리는 건 비추**(UX·브랜드 훼손).

### D. 30일 액션 (현실적)
1. **1주차**: 스레드 계정 세팅 + 매일 게시 시작(초안 7개로 바로 시작). 브리핑 랜딩 머지.
2. **2주차**: 어떤 글/훅이 도달 잘 나오는지 관찰 → 잘 되는 앵글로 발행 클러스터 조정.
3. **3~4주차**: 잘 먹힌 포맷 2배로. 팔로워 → 푸시 구독 전환 동선 강화.
4. 매주: **팔로워·구독자 수**만 기록(페이지뷰 무시).

## 4) ✅ 아침에 네가 결정할 것 (이것만 정하면 됨)

1. **콘텐츠 앵글 확정** — "오늘의 이슈, 돈으로 읽다"(추천, 안전·온브랜드) vs 순수 핫토픽(연예/정치, 도달↑·리스크↑). 정하면 스레드 보이스 그에 맞게 조정.
2. **스레드 자동 게시 연결?** — 지금은 *드래프트만* 생성(외부 게시 안 함). 진짜 자동 게시 원하면 **Threads/Meta API 토큰** 필요(계정 권한). 주면 자동 포스팅까지 연결. (아니면 복붙 운영)
3. **dev → main 머지(=배포) 승인** — 브리핑 랜딩·헤더 메뉴를 라이브로 올릴지. (위 1) 리뷰 후)

> 나머지(수익화 features, SEO 추가)는 위 결정 후 같이 진행 추천 — 지금 무인으로 막 건드리면 브랜드/레이아웃 리스크라 **문서로만** 정리함.
