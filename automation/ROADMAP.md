## 이것을 크라마에 대입하면 — 4단계

### 1단계. 측정 (지금 당장 — 안 보이면 못 고친다)
현재 GA4는 페이지뷰 수준. 근거 1·5를 적용하려면 **액티베이션·리텐션이 보여야** 함.
- [x] GA4 이벤트 계측: `save_item`(하트), `profile_save`(내 정보), `push_nudge`(노출/수락) — BaseLayout 공용 스토어에 심어 전 페이지 자동 (2026-07-05)
- [ ] **액티베이션 정의(가설)**: "첫 방문 세션에서 ①계산기 결과 1회 + ②하트 or 프로필 저장 1개" → 이 전환율을 주간 추적
- [ ] GA4 잔존 보고서(코호트)를 주 1회 확인하는 루틴 — 목표: **D7 ≥ 7%** (상위 25% 라인)
- [ ] Supabase 카운트 주간 리포트(user_profile·saved_items·push_subscriptions 증감) — 크론 스크립트

### 2단계. 리텐션 (근거 3·4 — 신규 유입보다 재방문 먼저)
이미 가진 무기: 데일리 브리핑, 출석 스트릭+밀웜, D-day 알림(세금4종·정책·복지·시험·예적금 만기), 알림 넛지.
- [ ] **스트릭 노출 확대**: 현재 /me에만 → 홈·브리핑 상단에 미니 스트릭(Duolingo: 스트릭 가시성 = 핵심 장치). 10일 마일스톤 보상 강화
- [ ] **푸시 = 제1 리텐션 채널**: 옵트인 퍼널 측정(넛지 노출→수락률). 금융 벤치마크(72%)에 크게 못 미치면 넛지 카피/타이밍 실험
- [ ] 브리핑 푸시 주기 점검: 주 1회 이상 수신자 리텐션 +440% 근거 → 데일리 푸시 구독을 온보딩·넛지에서 더 세게 제안
- [ ] 이탈 복귀 훅: D-day 알림이 자연 복귀 장치 — 저장 항목 없는 유저에게 '세금 캘린더 기본 알림 3종' 원탭 제안 (검토)

### 3단계. 유통 (근거 2·6 — 콘텐츠 추가보다 배관 공사)
- [ ] **네이버 서치어드바이저 등록 + 사이트맵 제출** ← 국내 모바일 검색 60%가 비어 있음. *(사용자 액션: 네이버 로그인 필요 — .env 요청 목록에 추가)*
- [ ] NerdWallet식 **토픽 클러스터 내부링크**: 계산기 ↔ 관련 글 ↔ 허브(/tax 같은) 양방향. 새 글 자동발행 파이프라인에 '관련 계산기 링크' 자동 삽입 검토
- [ ] **고의도 페이지 우선**: rates/loans/salary/tax류가 전환·수익의 코어(NerdWallet 교훈: 트래픽 양보다 의도). 이 페이지들의 검색 노출·체류를 우선 개선
- [ ] Threads 수동 배포 루틴 유지(주 3회, gen-cards 번들) — <1000명 구간은 유통이 콘텐츠보다 병목

### 4단계. PMF 검증 (근거 5 — 지표가 쌓이면)
- [ ] WAU 100+ 도달 시 Sean Ellis 설문("크라마가 사라지면?") 1문항 위젯 — 40% 기준으로 다음 투자처 결정
- [ ] 40% 미만이면: '매우 실망' 응답자의 공통 사용 패턴(어떤 기능?)에 집중, 나머지는 과감히 보류
- [ ] 수익화(제휴 대출/카드 비교 등)는 PMF 신호 이후 — 지금 붙이면 노이즈
